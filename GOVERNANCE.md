# Governance

Openflow is a founder-led open-source project.

## Decision Model

- Maintainers are responsible for roadmap and final technical decisions.
- Community input is welcome and considered in good faith.
- Not every feature request will be accepted.

## Contribution Path

- Small fixes can go directly through PR.
- Major features should be discussed first via issue/discussion.
- Experimental directions are encouraged via plugin or fork.

## Philosophy

Openflow AI is built for builders.

We value shipping over perfection, practical solutions over theoretical elegance, and respectful collaboration over endless debate.

Contributions are welcome, but the project maintains a strong product vision. Not every feature will be accepted. If you want to build something experimental, create a plugin or fork.
