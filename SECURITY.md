# Security Policy

## Reporting a Vulnerability

Please do not open a public issue for security vulnerabilities.

Report vulnerabilities privately to the maintainers with:

- A clear description of the issue
- Reproduction steps or proof of concept
- Potential impact
- Suggested mitigation (if any)

We will acknowledge receipt as soon as possible, investigate, and provide updates.

## Scope

This includes:

- Authentication and authorization flaws
- Secrets exposure
- Dependency vulnerabilities with realistic exploitability
- Data leakage and privacy issues

## Responsible Disclosure

Please allow reasonable time for investigation and remediation before public disclosure.
